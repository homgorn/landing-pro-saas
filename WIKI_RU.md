# 📐 Универсальная Wiki Landing Pro: Техническая и SEO Спецификация

Добро пожаловать в официальную техническую Wiki проекта **Landing Pro SaaS**. Этот документ определяет архитектурные стандарты и стратегии SEO-оптимизации, реализованные в ядре системы.

---

## 🏗 Архитектура системы

### 1. Мультиагентная оркестрация (MAO)
Движок использует слой суверенной оркестрации, определенный в `PipelineService.ts`. В отличие от стандартных решений, он следует **Линейному эволюционному паттерну**:
- **Экстракция Design DNA:** Использование Vision-моделей для квантификации эстетических параметров донора.
- **Контекстный синтез:** Слияние пользовательских промптов с данными поиска по нише (RAG).
- **Генерация артефактов:** Выдача чистого HTML-кода без зависимостей.

### 2. Спецификация Design DNA
Vision-агент извлекает объект `DesignSpec`:
```json
{
  "palette": { "background": "#...", "primary": "#..." },
  "typography": { "heading": "...", "body": "..." },
  "layout": "Grid/Flexbox centered"
}
```
Это гарантирует визуальное соответствие скриншоту-донору более чем на 95%.

## 📈 Проектирование SEO-производительности

### ⚡ Оптимизация Core Web Vitals
- **Lazy Loading:** Все генерируемые изображения используют `loading="lazy"`.
- **Preload Criticals:** Шрифты и Hero-ассеты предзагружаются для оптимизации LCP.
- **Zero CLS:** Фиксированные соотношения сторон для медиа-контейнеров.

### 🔍 Видимость в поиске
- **Семантический HTML5:** Строгое использование `<main>`, `<article>`, `<section>` и `<header>`.
- **JSON-LD Schema:** Каждая страница включает микроразметку для улучшенных сниппетов в Google.
- **Мета-теги:** Автоматическая генерация `og:image`, `twitter:card` и `canonical` тегов.
